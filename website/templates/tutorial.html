Tutorial


Welcome to the Web-BO tutorial page!

This guide will take you through all the steps to set up and run optimisations using Web-BO, including uploading data, defining experiments, adding measurements, and downloading results.


For further clarification, a video tutorial is available at the end of this guide, along with a frequently asked questions (FAQs) section.

Home page


From the Home Page you can navigate to various sections of Web-BO:

  • Explanations: Access tutorials, a case study, and additional explanations.

  • Datasets: Upload and manage your datasets.

  • Experiments: Set up new experiments and view those currently in progress.

  • Reset: Reset the database associated with your user.

Optimisation case study


We recommend that you start by exploring the optimisation case study provided. This will guide you through the basics of Web-BO using a sample dataset.

  1. Navigate to the case study:
    • Press the Explore! button on the home page, and read through the case study description.

  2. Add the sample-reizman-suzuki Dataset:
    • Scroll to the bottom of the case study pop-up.
    • Select 'Single Objective' or 'Multi Objective' to explore the correct dataset for your desired optimisation case study.
    • Your chosen dataset will be added to the Datasets area of the home page.

  3. Explore Dataset Features:
    • Navigate to the Datasets area of the home page and select the sample-reizman-suzuki dataset to load it.
    • Identify the different variables in the parameter space and understand their roles.

  4. Getting recommendations:
    • Select Setup your experiment.
    • Follow the methodology in section 5: Getting recommendations & adding measurements.
    • Note: Web-BO will show the measurements for you to submit, as you are not running the experiment.

Uploading a dataset


In the Datasets section, you can upload your own datasets to use in Web-BO.


Web-BO offers an example dataset in the csv format expected; more information can be found in the FAQs section.


  1. Navigate to the Datasets Section:
    • Select Add dataset from the home page.

  2. Choose upload options:
    • CSV File: Select your dataset CSV file from your computer and assign a unique name - see FAQs for formatting advice.
    • DataLab: Provide the relevant information to connect to your Data Lab account.

  3. Explore Dataset Features:
    • Navigate to the Datasets area of the home page and select the dataset to load it.
    • Web-BO will display a breakdown of dataset details, including the variables and summary statistics.

Setting up your experiment


To start an optimisation, set up an experiment using your uploaded dataset:

  1. Navigate to Experiments:
    • Go to the Experiments section of the home page and select Add Experiment.

  2. Number of Objectives:
    • Choose to create a single-objective or multi-objective optimisation experiment.
    • For a multi-objective optimisation experiment, you will be prompted to enter the number of targets you wish to optimise.

  3. Dataset Selection:
    • Choose a dataset from those you have uploaded, give your experiment a unique name, and click Submit Dataset.

  4. Parameter Space:

    Define the variable type and include additional requested information:

    • Categorical: Requires an uploaded csv file - see FAQs section for more details.
    • Integer: Define a range (min. and max. values).
    • Continuous: Define a range (min. and max. values).

  5. Hyperparameter Configuration:
    • GP Kernel Type: Choose between Matern and Tanimoto.
    • Acquisition Function: Select Expected Improvement or Probability of Improvement.
    • Batch Size: Input the number of experiments to run before recieving new recommendations from Web-BO - see FAQs section for more details
    • Target Variable(s): Select the objective variable(s) you want to optimise - see FAQs section for more details.
    • Optimisation Type(s): Indicate if you want to Maximise or Minimise the target variable(s).
    • Relative Weight (for multi-objective optimisations only): Specify the relative importance of the targets - see FAQs section for more details

  6. Finalize Setup:
    • Click View experiment to create the experiment.

Getting recommendations & adding measurements


Now that your experiment is set up, start the optimisation by generating recommendations and adding measurements.

  1. Get Next Recommendation:
    • Click Get Next Recommendation. Web-BO will suggest values for each variable based on the selected target. The first recommendation (Iteration 1) represents the first trial.

  2. Run Experiment & Add Measurements:
    • Conduct the experiment using the suggested conditions from Web-BO.
    • Click Add Measurements:
      • Input the exact conditions used.
      • Enter the observed target value at these conditions and click Submit Measurements.

  3. Repeat Process:
    • Continue selecting Get Next Recommendation and Add Measurements after each experimental trial until the target is optimised to your satisfaction.

Downloading results


Once the optimisation process is complete, download the final dataset.

  • Scroll to the bottom of the page and click Download Dataset:
    • The file will be saved to your computer’s download folder.

Frequently Asked Questions (FAQs)


Where can I access the csv toy dataset?

Web-BO offers an example dataset in the csv format expected; this is contained in the toy-dataset directory of the Web-BO GitHub (https://github.com/austin-mroz/webBO).

  • A csv file outlining the options for the categorical variable is included (required for defining the parameter space within experimental set-up).
  • In order to use the toy dataset, these files must first be downloaded locally and then uploaded using the csv upload feature in Web-BO

What is the required format for a input csv dataset?

To get the correct file format it is easiest to write your data in an excel spreadsheet, and then save it as a ".csv" file. An example input file (for a dataset) is shown on the right. There are three things to note:

  • The file starts in cell A1, with no empty rows or columns above/next it. This ensures that when your data is tabulated there are no empty cells.
  • Row 1 will be your column headers in Web-BO, the rows after that your data entries, and each columns is a different variable.
  • Row 9 in the image should not be part of your csv file, it is there to show the type of each variable

How do I upload a dataset from my Datalab account?

To Upload a dataset from your Datalab account, select Datalab as your file upload option.

  • You will be required to enter the details of your Datalab account (API Key, Domain, Collection ID, Block ID).
  • Variables must be indicated in the general format: 'Variable 1, Variable 2, ...'

If you require further guidance on connecting to a Datalab account, reach out to a member of the Web-BO team.

What input is required when setting up an experiment involving a categorical variable?

When defining the parameter space (within experimental setup), a csv file is required to describe the options that comprise the optimisation space.

  • To get the correct file format it is easiest to write your data in an excel spreadsheet, and then save it as a ".csv" file.
  • This csv file should take the general format shown on the right, for the categorical variable of solvents.

Selecting targets: What types of variables can be optimised?

  • Web-BO can only optimise numerical targets.
  • This means that only variables defined as integer or continuous variables in the parameter space can be optimised.

What is batch size?

  • Batch size describes the number of experiments you wish to run before recieving new recommendations from Web-BO.
  • For example, a batch size of 2 means Web-BO will provide 2 sets of initial conditions to test, then, based on those experimental results, it will generate the next 2 sets.

What is relative weight?

  • Relative weight describes the relative importance of the targets for a multi-objective optimisation experiment.
  • Weight values must be positive numbers and will be normalised, so only their relative magnitudes matter.
